# Design Components

Use this file to record durable component-level design specifications that agents can implement.

## Component Template

```md
## Component Name

**Purpose:** What the component does.

**Variants:** Sizes, states, visual variants.

**Behavior:** Interaction details.

**Accessibility:** Keyboard, screen-reader, focus, contrast notes.

**Tokens:** Relevant entries from `design/tokens.json`.

**Implementation notes:** Optional technical guidance.
```
