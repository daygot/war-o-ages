# First Project Fill-In Checklist

Use this after creating a fresh project copy from the template.

## 1. Project Definition

Update `context/PROJECT.md`:

- Project name
- Purpose
- Users/audience
- Product goals
- Non-goals
- Success criteria
- Key links

## 2. Current State

Update `context/CURRENT.md`:

- Active goals
- Active work
- Blockers
- Next recommended actions

## 3. Agent Setup

Update `agents/registry.md`:

- Human collaborators
- Agent IDs
- Device names
- Default roles

Update `agents/status.md` with initial active/idle states.

## 4. Design Setup

Update `context/DESIGN.md` and `design/tokens.json`:

- Design goals
- Brand/mood
- UX principles
- Initial color/type/spacing tokens

## 5. Architecture Setup

Update `context/ARCHITECTURE.md`:

- Chosen stack
- Major components
- External services
- API/data contracts
- Security/privacy notes

## 6. First Tasks

Create task briefs from `tasks/TEMPLATE.md` and place them in `tasks/backlog/`.

Recommended first tasks:

- Define MVP scope
- Choose stack/tooling
- Create initial design direction
- Scaffold app/project structure
- Create first vertical slice
